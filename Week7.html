Functions are first-class objects - passed around like every other value.

Functions have their own properties and methods and can accept other functions as parameters and can be returned by other functions.

  • Function properties and methods
  • All functions have a length() method that reveals how many parameters that function has.
  • Functions have a call() method that can be used to set the value of this inside a function. Such as sayHello.call(clark) returning the message from sayHello with the name now equal to the other variable.
  • This method can also be used to change a parameter - change Hello to How do you do.
  • This is always the first argument, so any parameters needs to come second.
  • You can customize your own properties inside functions.
  • Memoization is useful to provide result caching - this is great if a function takes a while to compute a return value.
  • Immediately Invoked function expressions
  • AKA iffy (IIFE) is anonymous
  • Invoked as soon as it’s defined.
  • Perform task and keep variables wrapped in scope
  • Self-defining functions
  • (Functions that define and rewrite themselves).
  • JavaScript lets you define, call, and redefine a function within itself.
  • Assign an anonymous function to a variable that has the same name as the function.
  • Recursive functions
  • One that invokes itself until a condition is met.
  • Great for iterative processes.
  • Function that calculates the factorial of a number;
  • Callbacks
  • Used to facilitate event-driven asynchronous programming.
  • JavaScript is single-threaded. - Callbacks allow the code to run out of order or asynchronously.
  • Callbacks ensure that waiting for tasks to complete doesn’t hold back execution.
  • Promises
  • Promise represents the future result of asynchronous operation.
  • Help simplify the process.
  • Avoids convoluted code.
  • Async functions
  • Added to the ES2017 spec.
  • Allows you to write asynchronous code as if it was synchronous.
  • Wrap the return value of a function in a promise.
  • Functions that return functions
  • Functions can accept another function as an argument but they can also return a function.
  • Closures
  • Difficult to wrap your head around.
  • Function scope - value of variable only available inside block it was created if let or const is used. (Including the body of a function if the var keyword is used).
  • HEY! I had an issue with this in my Web Engineering JavaScrip calculator. The variable was originally declared within an if() method but then later used outside the method for something else. I didn’t run into an issue until I tried to validate it.
  • Returning Functions - A closure is formed when the inner function is returned by the outer function.
  • Introduction to functional programming
  • Programming paradigm - including object oriented and procedural.
  • JavaScript is a multi-paradigm language
  • Currying
  • Partial application of functions. (Names after Haskell Curry - not actual curry).
  • Function is said to be curried when not all arguments have been supplied to the function - so it returns another function that retains the already provided arguments.
  • Relies on higher-order functions.
  • power() function is an example of this.

Ajax

  • Clients and servers
  • The internet is divided into two parts - clients and servers.
  • Server is required when requesting resources.
  • PHP, Ruby, NODE.js, or .NET are server side languages (ohhhhhh).
  • A brief history of Ajax
  • In 1999 by Microsoft implementing the XMLHTTP ActiveX control in Internet Explorer 5. (Who uses Internet Explorer anymore, holy cow).
  • Asynchronous loading techniques started to be noticed by Google (when Gmail and google maps actually became a thing in the early 2000s).
  • Ajax was coined by Jesse James Garrett in 2005. Asynchronous, JavaScript, XML,
  • Communicating with the server using the Fetch API
  • XMLHttpRequest object was standardized by the WHATWG and W3C as part of HTML5 specs.
  • Superseded by FetchApi
