CSS3 Transforms and Transitions

CSS3 can be used to tilt, scale, move, and flip elements without having to use JS or JPEGs.

TRANSFORMS
Transform functions manipulate an element’s appearance. Requires vendor prefixing for IE9, Android up to 4.4.3, iOS8, and Blackberry 10.

TRANSLATION

Translation functions move the element left, right, up, or down. - similar to position: relative. Translation moves the element without impacting the flow of the document. Translated elements can only be moved relative to current position.

Translatex and translatey functions move the element vertically and horizontally respectively.

SCALE

Scale function scales an element by the defined factors horizontally then vertically (x, y); If only one value is provided it’s applied to both x and y. Providing just one value keeps the aspect ratio in proportion - so the image isn’t distorted. scaleX and scaleY can be used to only effect the horizontal or the vertical. The center of the element will remain the same unless you include the transform-origin property. Scaling has no impact on document flow so it might mess things up in an inline-block - you might consider adjusting the height, width, or font-size instead of scale().

ROTATE

Rotate functions rotate an element around the point of origin by a specified angle value. Point of origin is defaulted to the elements center. Declare rotate before the translate so it’s applied first.

SKEW

Skew specifies a skew along the x and y axes.

TRANSFORM-ORIGIN

By moving the origin of the rotation, you can get more effects.

TRANSLATE BEFORE ROTATING.

TRANSITION

Allows for CSS properties to change over time. Essentially providing simple animations. Declared along with the regular styles on an element - pretty sure I used this to change the background color of a website between pink and purple...

Transition-duration controls how long the transition takes. .2s (or 200ms) is optimal for smooth transitions that are still apparent.

Transition-timing-function controls the pace even further (start slow then move fast?).

Ease - slow start, speeds up, ends slowly.

Linear - constant speed.

Ease-in - slow start, speeds up, ends abruptly.

Ease-out - starts fast, slows progressively.

ease-in-out. - accelerates more sharply in the beginning.

http://estelle.github.io/animation/files/cubicbezierprint.html is a great website to play with writing your own timing functions.

Transition-delay can be used to be sure the user is deliberately hovering over the element - 50ms is a good speed for this.

Transition can be condensed to one command but if a delay is included, you must also include a duration, and the duration must precede the delay.

Canvas, SVG, and Drag and Drop

CANVAS

Text between canvas tags will only be shown if the canvas element is not supported in the current browser. You have to give it an id and a class so you can grab it from the DOM.

Width and height attributes must also be set. - defaults is 300x150, DON’T use CSS because it only determines how big the box is, not the canvas’s coordinate system.

If you want to visually differentiate between canvas and the rest of the webpage you have to give it a border - which you can do in CSS.

To grab hold of the canvas = var canvas = document.getElementById("myCanvas"

Set the canvas.getContext() to a local variable and pass in the String parameter “2d”.

Strokestyle and fillStyle properties are set on a context object. Both take one of three values - string for color, cavasGradient object, or canvasPattern object.

fillRect and strokeRect take coordinates of where we want to begin drawing and filling the rectangle. Top left corner is (0,0)

You can also use patterns to fill in the rectangle, and you can use different commands to get different shapes.

You can convert to greyscale, manipulate images, and display text on canvas.

SVG - SCALABLE VECTOR GRAPHICS

A file format that lets your describe vector graphics using XML. eXtensible Markup Language. A system meant to annotate text. XML can be used to describe data.

You can use an image editor to help create SVG images rather than coding it by hand.

DRAG AND DROP

This API allows us to specify which elements are draggable. And to dictate what should happen when those draggable elements are dragged over or dropped onto other elements - sound like how you would create a jig saw puzzle game.
